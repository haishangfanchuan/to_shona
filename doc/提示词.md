## 游戏概念

### 名称
「街角的话」

### 核心体验
两个人傍晚在街边沉默地散步，玩家需要保持和对方同步的步伐。走过城市街道，最终在地铁站门口坐下，一句一句说出道歉的话，然后等待对方的选择。

### 故事线
> 傍晚，两个人沉默地走在街道上。谁都没开口。
> 走过熟悉的眼镜店，路过常去的零食店。
> 夜色降临，城市灯火亮起，地铁站就在前方。
> 在台阶上坐下，终于把一直想说的话，一句一句说了出来。
> 然后，等待对方的回应。

### 两幕结构

#### 第一幕：傍晚街道「沉默」
- 色调：暖橙 → 渐暗
- 卷轴长度：800px
- 两人初始间距：50px，同步状态下缓慢收缩至 20px
- 两个回忆触点：眼镜店（x=350）、零食店（x=600）
- 走到触点附近时头顶浮出 `!` 气泡，点击弹出回忆面板（文案可自定义）
- 主角头顶持续显示 `...` 气泡（三个点依次跳动动画）

#### 第二幕：夜色城市 → 地铁站「说出口」
- **前半段（CityNightScene）**：夜色城市行走，卷轴 500px，两人并肩（间距 10-20px），远处地铁标志逐渐放大
- **后半段（SubwayScene）**：到达地铁站后自动过场——两人走到台阶坐下。进入对话模式：每次点击屏幕，一句道歉的话从主角头顶浮出并飘到屏幕上方固定位置。所有话说完后，汇聚成一封完整的信呈现在屏幕上。然后出现两个选择按钮。

### 核心机制

#### 操控方式
- 左右滑动或虚拟摇杆控制主角水平移动
- NPC（对方）以固定缓慢速度自动向右行走

#### 步伐同步系统
- 玩家太快（超前 NPC 超过 60px）：NPC 停下低头，温度条冻结
- 同步（距离在 -40px 到 60px 之间）：温度条上升，画面变暖
- 玩家太慢（落后 NPC 超过 40px）：屏幕四角暗角效果加深，温度条下降

#### 温度条
- 屏幕顶部横向进度条，左端冷右端暖
- 温度值影响最终画面：>80 满天繁星+大爱心，50-80 少量星星，<50 几乎无星

#### 回忆触点
- 两人都走到触点 ±20px 范围内时触发
- 触发回忆后温度条额外 +5

### 结局分支

#### 结局A：不原谅（点击「算了」）
完整的 12 秒演出时间轴：
- 0-0.5s：场景淡入，两人坐着，信件文字显示
- 0.5-1.0s：信件文字淡出
- 1.0-1.5s：切换为独立精灵，NPC 头顶浮出 `...`（犹豫）
- 1.5-2.0s：NPC 播放起身动画（2帧）
- 2.0-2.5s：NPC 站立停顿 0.5s
- 2.5-6.0s：NPC 背影行走离开（速度为正常的一半），主角不动，头顶 `...`
- 6.0-7.0s：NPC 渐隐消失（走入地铁）
- 7.0-8.0s：主角独坐画面停留
- 8.0-9.5s：画面和 BGM 同步淡出至黑
- 9.5-10.0s：纯黑屏静默
- 10.0-11.0s：屏幕中央淡入一行白色文字「也许下次能说出口」
- 11.0s 起：永久停留

#### 结局B：原谅（点击「好吧」）
待定，暂用占位方案：两人坐着 + 爱心浮现 + 星空粒子（根据温度值决定密度）

### 技术约束
- 游戏引擎：Phaser 3（3.80+）
- 语言：TypeScript
- 构建工具：Vite
- 音频：Howler.js
- 游戏逻辑分辨率：192 x 320（竖屏）
- 像素渲染：pixelArt: true, antialias: false, roundPixels: true
- 背景完全依赖贴图，不在代码中绘制背景元素
- 角色精灵尺寸：32x32
- 调色板限定 32 色以内

### 场景流转
```
BootScene → TitleScene → StreetScene → CityNightScene → SubwayScene → EndingA/EndingB
```
场景间通过淡出/淡入过渡（500ms），温度值和间距在场景间传递。
