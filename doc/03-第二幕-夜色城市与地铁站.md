# 03-第二幕-夜色城市与地铁站.md

## Part A: CityNightScene (夜色城市)

### 场景参数表

| 参数名 | 设定值 |
| :--- | :--- |
| **场景卷轴长度** | 500px |
| **环境光效** | 较暗背景，部分霓虹灯窗户常亮 |
| **地铁标志 (背景透视)** | X=400 处开始，Scale 从 0.5 逐渐放大至 1.0 |
| **初始化间距** | 继承第一幕传递的值 |

### 核心逻辑
本段旨在情绪收拢。
* 玩家无法大幅度拉开距离，最大速度受到限制。
* **透视放大效果**：当地铁站标志 (单独一个 Sprite 层) 进入视野时，根据玩家的 X 坐标计算比例：
    ```typescript
    let subwaySignDist = 500 - player.x;
    if (subwaySignDist < 200) {
        subwaySign.setScale(1.0 - (subwaySignDist / 200) * 0.5);
    }
    ```
* 过渡：达到 X=500，直接使用 `fadeOut(500)` 进入 `SubwayScene`，不改变传递的数据。

## Part B: SubwayScene (地铁站)

### 场景参数表

| 参数名 | 设定值 |
| :--- | :--- |
| **场景类型** | 静态单屏无卷轴 |
| **主角坐下坐标** | `x: 60, y: 260` |
| **NPC坐下坐标** | `x: 120, y: 260` |
| **信件文字最终Y坐标**| 从 `Y: 40` 开始向下排列 |

### 坐下过场动画时间轴 (入场演出)
* **0ms**: 画面从黑屏淡入，两人从左侧走入屏幕。
* **1000ms**: 走到指定坐标，停止行走动画。
* **1500ms**: 两人同步播放 `npc_sit_stand.png` 帧 1 (弯腰)。
* **1800ms**: 两人同步播放帧 2 (坐下)，保持该状态。
* **2500ms**: 界面显示 UI 提示文字："点击屏幕说出口"，并闪烁。

### 对话模式流程 (lineIndex 驱动)
取消摇杆控制，全屏监听 `pointerdown`。
维护变量 `let lineIndex = 0;`

```typescript
function onPointerDown() {
    if (lineIndex >= DIALOGUES.SUBWAY_APOLOGY.length) return;

    let textStr = DIALOGUES.SUBWAY_APOLOGY[lineIndex];
    let startX = player.x;
    let startY = player.y - 30; // 头顶位置

    // 创建文字对象
    let textObj = this.add.text(startX, startY, textStr, {
        fontSize: '10px', fill: '#FFFFFF', alpha: 0
    });

    // 计算目标位置 (上方信件区域，逐行排列)
    let targetY = 40 + (lineIndex * 15);
    let targetX = 96; // 屏幕中心居中
    textObj.setOrigin(0.5, 0.5);

    // 飘移动画 (Tween)
    this.tweens.add({
        targets: textObj,
        x: targetX,
        y: targetY,
        alpha: 1,
        duration: 1500,
        ease: 'Sine.easeOut'
    });

    lineIndex++;

    if (lineIndex === DIALOGUES.SUBWAY_APOLOGY.length) {
        triggerFinalPresentation();
    }
}
```

### 最终选择界面呈现
触发 `triggerFinalPresentation()` 后：
1. 等待最后一句文字飘浮到位。
2. 整个文字区块组合应用轻微的呼吸动画 (Scale 0.98 <-> 1.02，时长 2000ms 循环)。
3. 根据当前温度值判定：
   * `Temp > 80`: 背景出现明显的光晕。
   * `Temp < 50`: 画面四周维持较深暗角。
4. 延迟 1500ms 后，淡入两个选择按钮。

### 按钮行为
* **算了 (Reject)**：点击后调用 `this.scene.start('EndingScene', { result: 'reject' })`。
* **好吧 (Accept)**：点击后调用 `this.scene.start('EndingScene', { result: 'accept', temp: currentTemp })`。
